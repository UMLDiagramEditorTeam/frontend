# UML Diagram Editor

Веб-приложение для разработчиков - графический редактор, упрощающий переход от проектного моделирования к написанию кода за счёт автоматической генерации программных заготовок (скелетонов)

Приложение предоставляет пространство для создания, сохранения и редактирования UML-диаграмм через drag-and-drop интерфейс. Киллер-фича - генерация кода: при создании диаграммы классов система автоматически формирует структуру файлов на Python или Java (классы, атрибуты, методы, модификаторы доступа), а для диаграмм последовательностей генерирует скелеты методов с логикой вызовов между объектами.

## Стек

- **React** + **TypeScript** - клиентское приложение
- **Vite** - сборщик и dev-сервер
- **React Router** - маршрутизация
- **Ant Design** - библиотека UI-компонентов
- **Axios** - HTTP-клиент
- **ESLint** - статический анализ и форматирование
- **Docker** + **nginx** - контейнеризация и раздача приложения

Архитектура построена по методологии **FSD** с разделением на слои `app`, `pages`, `features`, `shared`

## Реализовано

- Drag-and-drop редактор UML-диаграмм классов и последовательностей
- Создание, сохранение и редактирование проектов
- Экспорт диаграмм в графические форматы (PNG, SVG)
- Аутентификация с автоматическим обновлением токенов
- Типобезопасная интеграция с API на основе OpenAPI-спецификации

## Запуск проекта

### Для разработки

```bash
npm install
npm run dev
```

приложение будет доступно на `http://localhost:5173`

### Сборка проекта

```bash
npm install
npm run build
```

Предпросмотр прод-сборки локально:

```bash
npm run preview
```

## Переменные окружения

Перед запуском создайте файл `.env` в корне проекта:

```
VITE_API_URL=/api/v1
```

Для локальной разработки с отдельно поднятым бэкендом можно указать полный адрес, например `VITE_API_URL=http://localhost/api/v1`

## Запуск через Docker

Приложение упаковывается в Docker-образ: nginx раздаёт собранную статику и проксирует запросы `/api/` на бэкенд,
наружу открыт только порт **80**

```bash
docker compose up --build
```

после запуска:

- `http://localhost` - приложение
- `http://localhost/api/docs` - Swagger-документация API

## Линтер и формат

Проверка линтером:

```bash
npm run lint
```

Форматирование:

```bash
npm run format
```

Pre-commit хуки:

```bash
npm run prepare
```

## Генерация API-клиента

Типы и SDK клиента генерируются автоматически на основе OpenAPI-спецификации бэкенда:

```bash
npm run generate:api
```
